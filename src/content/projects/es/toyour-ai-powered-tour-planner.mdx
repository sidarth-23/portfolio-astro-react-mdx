I came across a really interesting Figma template for a chat-based travel planner and thought — why not build it out? What started as a quick implementation exercise turned into a fun exploration of a UI design trend I had been wanting to try for a while: **neobrutalism**.  
 Here’s a look at how the project came together, what I used, and why I’m kind of obsessed with neobrutalism now.

---

## What is Toyour?

**Toyour** is a chat interface that lets users ask for travel tour suggestions and browse curated itineraries. Think of it as a conversational travel concierge — you type in something like _“I’m planning a 4-day trip to Japan, prioritise Tokyo and Kyoto”_ and the UI responds with a structured list of places, each clickable for more detail.

The data is all mocked for now, but the UX flow is real: a chat window on one side, a detail panel on the other, and a clean toggle for light/dark mode.

---

## The Stack

- **Framework:** Next.js 15 (App Router)
- **Language:** TypeScript
- **UI Components:** shadcn/ui — neobrutalism theme
- **Package Manager:** Bun
- **Styling:** Tailwind CSS
- **Deployment:** Vercel Nothing exotic. The whole point was to move fast and focus on the visual result.

---

## Why Neobrutalism?

Neobrutalism is a UI design style that leans into raw, intentional design choices — thick borders, hard drop shadows, flat bold colours, and a slightly “unpolished” aesthetic that somehow looks incredibly deliberate. It’s the anti-Material-Design. No soft shadows, no blurred glassmorphism — just shapes with edges.

Shadcn recently added a **neobrutalism variant** to its component library and it’s genuinely great to use. You get all the accessibility and composability of shadcn’s components, but with a visual personality that actually stands out.  
 For a travel app that’s supposed to feel lively and adventurous, neobrutalism felt like the right call over yet another minimal white-card layout.

> 💡 **Tip:** You can switch to the neobrutalism theme in `components.json` when initialising shadcn in your Next.js project.

## Project Structure

```plaintext
src/
├── app/
│   ├── page.tsx          # Main layout — chat + detail panel
│   └── layout.tsx        # Root layout with theme provider
├── components/
│   ├── chat/             # Chat window, message bubbles, input
│   ├── detail/           # Place detail panel
│   └── ui/               # shadcn neobrutalism components
└── data/
    └── mockData.ts       # Hardcoded trip & place data
```

The layout splits into two zones: the **chat panel** (where the conversation lives) and the **explore panel** (where selected place details render). Clicking a place name in the chat response populates the right panel — a simple but satisfying interaction.

---

## The Mock Data

Since there’s no backend, all the “AI” responses are pre-written and triggered by matching input patterns. The Japan itinerary is the demo scenario:

- **Tokyo:** Go-Karting in Shibuya, Meiji Shrine + Food Tour
- **Kyoto:** Tea Ceremony in Gion, Arashiyama Bamboo Forest, Fushimi Inari Shrine at Sunset  
Each place has a title, description, and an image that loads in the detail panel. It’s a small dataset but enough to demonstrate the full interaction loop.

## What I’d Add Next

This was a quick project — a weekend build to scratch the itch of trying neobrutalism in a real layout. But if I were to take it further:

- **Real AI backend** — hook it up to an LLM via an API route in Next.js
- **More destinations** — expand the mock data or pull from a travel API
- **Animated transitions** — the panel swap feels like it could benefit from a smooth slide-in
- **User history** — persist chat sessions in localStorage or a simple DB
- **Mobile layout** — the current layout is desktop-first; a stacked mobile view would be a good next step

---

## Final Thoughts

Sometimes the best projects come from just picking up a Figma template and asking _“what would this actually look like if I built it?”_ Neobrutalism made the whole thing feel fresh — and shadcn’s component library made it surprisingly fast to implement without sacrificing quality.

If you’ve been sitting on a design inspiration and haven’t built it yet, this is your sign. The stack is boring in the best way, the aesthetic does the heavy lifting, and you’ll have something deployed in a day.
