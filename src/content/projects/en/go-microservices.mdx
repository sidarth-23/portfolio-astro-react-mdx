## Overview

Building microservices in Go shouldn't require reinventing the wheel every time. This toolkit provides the foundational pieces you need: service discovery, circuit breakers, health checks, and distributed tracing — all wired together with sensible defaults.

## Features

- **Service Discovery**: Automatic registration with Consul
- **Circuit Breaker**: Prevent cascade failures with configurable thresholds
- **Distributed Tracing**: OpenTelemetry integration out of the box
- **Health Checks**: HTTP and gRPC health endpoints
- **Graceful Shutdown**: Handle SIGTERM properly with timeout configuration

## Architecture

The toolkit follows a layered approach:

1. **Transport Layer** (HTTP/gRPC)
2. **Service Layer** (Business Logic)
3. **Repository Layer** (Data Access)
4. **Infrastructure** (Logging, Metrics, Tracing)

## Getting Started

```go
import "github.com/sidarth-23/go-microservices"

svc := micro.NewService(
    micro.Name("user-service"),
    micro.Version("1.0.0"),
)

svc.Init()
svc.Run()
```
