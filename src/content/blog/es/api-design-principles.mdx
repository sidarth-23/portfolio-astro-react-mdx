## Empieza por el Consumidor

Las mejores APIs se diseñan de afuera hacia adentro. Antes de escribir un solo endpoint, define qué necesita lograr el consumidor.

## Estrategia de Versionado

**Versionado por URL** (`/v1/users`) es simple y explícito. **Versionado por Header** es más limpio pero más difícil de depurar. Elige uno y mantente consistente.

## La Consistencia es Rey

- Usa sustantivos en plural (`/users`, no `/user`)
- Estandariza las respuestas de error
- Implementa paginación en cada endpoint de lista
- Soporta filtrado, ordenamiento y selección de campos

## Manejo de Errores

```json
{
  "error": {
    "code": "INVALID_INPUT",
    "message": "El formato del email es inválido",
    "field": "email"
  }
}
```

Incluye siempre:
- Código de error legible por máquinas
- Mensaje legible por humanos
- Contexto (campo, ID de solicitud)

## Rendimiento

- Usa headers de caché (ETag, Cache-Control)
- Implementa compresión de solicitud/respuesta
- Considera GraphQL para requisitos de datos complejos
- Limita la tasa por usuario Y por IP

## Documentación

Tu API es tan buena como su documentación. Usa OpenAPI/Swagger y mantenlo sincronizado con el código.

## Conclusión

Las grandes APIs son dogmáticas, consistentes y bien documentadas. Invierte tiempo desde el principio — tu yo del futuro te lo agradecerá.
