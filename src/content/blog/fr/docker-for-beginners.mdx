## Pourquoi Docker est Important

Les jours de "ça marche sur ma machine" sont révolus. Docker standardise votre environnement du développement à la production, éliminant toute une catégorie de bugs de déploiement.

## Concepts Clés

**Les images** sont des modèles. **Les conteneurs** sont des instances en cours d'exécution. **Les volumes** persistent les données. **Les réseaux** connectent les conteneurs.

## Builds Multi-Étapes

Le plus grand avantage pour les développeurs Go: compiler dans une image de build complète, puis copier le binaire dans une image runtime minimale.

```dockerfile
FROM golang:1.22-alpine AS builder
WORKDIR /app
COPY . .
RUN go build -o main .

FROM alpine:latest
COPY --from=builder /app/main .
CMD ["./main"]
```

Cela réduit la taille de l'image de ~300MB à ~20MB.

## Compose pour le Développement Local

```yaml
services:
  api:
    build: .
    ports:
      - "8080:8080"
    depends_on:
      - db
  db:
    image: postgres:16
    environment:
      POSTGRES_DB: myapp
```

## Conseils pour la Production

- Utilisez des tags d'image spécifiques, jamais `latest`
- Exécutez en tant qu'utilisateur non-root
- Analysez les images pour les vulnérabilités
- Utilisez des health checks

## Conclusion

Docker n'est pas seulement pour le déploiement — c'est un super-pouvoir de développement. Commencez à conteneuriser vos services dès aujourd'hui.
