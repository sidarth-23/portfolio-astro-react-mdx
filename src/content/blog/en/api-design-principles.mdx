## Start with the Consumer

The best APIs are designed from the outside in. Before writing a single endpoint, define what the consumer needs to accomplish.

## Versioning Strategy

**URL versioning** (`/v1/users`) is simple and explicit. **Header versioning** is cleaner but harder to debug. Pick one and stick to it.

## Consistency is King

- Use plural nouns (`/users`, not `/user`)
- Standardize error responses
- Implement pagination on every list endpoint
- Support filtering, sorting, and field selection

## Error Handling

```json
{
  "error": {
    "code": "INVALID_INPUT",
    "message": "Email format is invalid",
    "field": "email"
  }
}
```

Always include:
- Machine-readable error code
- Human-readable message
- Context (field, request ID)

## Performance

- Use caching headers (ETag, Cache-Control)
- Implement request/response compression
- Consider GraphQL for complex data requirements
- Rate limit by user AND by IP

## Documentation

Your API is only as good as its docs. Use OpenAPI/Swagger and keep it in sync with code.

## Bottom Line

Great APIs are opinionated, consistent, and well-documented. Invest time upfront — your future self will thank you.
